# Codex Instructions for Dream Runner

## Project

Dream Runner is a browser-based third-person dream-running game built in Godot 4.

## Role of Codex

Act as a senior Godot game developer and technical partner.

Prioritize:

1. Simple, working code
2. Clear folder structure
3. Small incremental changes
4. Browser performance
5. Readable scripts
6. Stable movement feel

## Development Rules

- Do not build huge systems in one step.
- Work in small commits.
- Explain what files you changed.
- Prefer simple Godot-native solutions before adding complexity.
- Avoid unnecessary plugins.
- Keep browser export in mind.
- Use placeholder assets before final assets.
- Movement feel is more important than visual polish at the start.

## Current MVP

Create a playable prototype with:

- third-person player controller
- high-speed running
- high jump
- smooth camera follow
- simple test terrain
- browser-export-friendly structure

## Technical Preference

Engine: Godot 4  
Language: GDScript  
Target: Browser/Web first
